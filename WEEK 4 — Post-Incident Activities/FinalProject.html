Response & Resolution

The IRT successfully managed containment, eradication, and simulated recovery by following the incident response plan.

5.1 Containment

  • **Endpoint Isolation:** Infected host `whoami` immediately isolated from the network.
  • **Perimeter Blocking:** Malicious URL and C2 IPs blocked at the firewall.
  • **Email Quarantine:** All instances of the phishing email were quarantined.

5.2 Eradication

  • **Malware Removal:** `safe-ransom.ps1` script securely located and removed.
  • **Persistence Checks:** Scanned for scheduled tasks or registry keys.
  • **Email Removal:** Malicious phishing emails deleted from all mailboxes.

5.3 Recovery

  • **Restoration:** Affected system restored from a known clean, pre-infection backup.
  • **Verification:** System fully scanned, verified clean, and reconnected to the network.
  • **Filter Updates:** All IoCs added to EDR, SIEM, and proxy blocklists.

Remediation Plan

The following analysis identifies the core failures and provides a mandatory action plan to enhance security posture and prevent recurrence.

6.1 Root Cause & Vulnerabilities

Vulnerability Phase of Attack Impact
Email Authentication Deficiency Initial Access Allowed spoofed email to bypass perimeter defense.
Endpoint Execution Control Execution Allowed PowerShell (`.ps1`) to run and encrypt files.
User Awareness Gap Initial Access & Execution User was successfully targeted, initiating the attack chain.

6.3 Final Recommendations for Remediation

  1. Implement DMARC/SPF/DKIM: Enforce strong email authentication to eliminate sender spoofing.
  2. Mandatory Awareness Training: Institute continuous phishing and social engineering training.
  3. Enhance Filtering & Sandboxing: Automate blocking of malicious URLs and enforce sandboxing of attachments.
  4. Enforce Multi-Factor Authentication (MFA): Implement MFA for all critical services to mitigate credential compromise.
  5. Strengthen Monitoring (EDR/SIEM): Deploy an EDR solution to block malicious script execution and enhance detection.
  6. Implement Security Baselines: Enforce least-privilege principles to limit malware impact.

6.4 Security Advisories for Employees

Scrutinize Sender Identity ‚ñ∫

**Verify the authenticity** of the sender's email address by checking for subtle misspellings, mismatched domains, or unusual sender behavior. Do not rely solely on the display name.

Proactive Link Validation ‚ñ∫

**Hover over all links** to inspect the destination URL before clicking. If the link is suspicious (e.g., unfamiliar IPs like `192.168.159.130`), **do not proceed**.

Challenge Urgent Requests ‚ñ∫

Treat any email demanding immediate action, especially involving account verification or financial details, as high-risk. **Independently verify** all such requests through a trusted, out-of-band communication channel (e.g., phone call).

Utilize Reporting Channels ‚ñ∫

**Immediately report** any suspicious or confirmed phishing email via the designated corporate reporting tool. Timely reporting is critical for organization-wide containment.

Safeguard Credentials ‚ñ∫

Never share Multi-Factor Authentication (MFA) codes, PINs, or passwords with anyone, including personnel claiming to be from IT Support.
